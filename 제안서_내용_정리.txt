## 1. 프로젝트 개요

### 1.1 프로젝트명
개인 맞춤형 뉴스 요약 및 TTS 서비스 (Project TNP - Today's News Personalized)

### 1.2 프로젝트 목표
사용자의 관심분야를 기반으로 개인화된 뉴스를 제공하고, AI 요약 및 음성 변환 기능을 통해
효율적인 뉴스 소비 환경을 제공하는 종합 뉴스 플랫폼 개발

### 1.3 기술 스택
- Backend: Spring Boot 3.4.5, Java 21, Spring Security 6.x
- Database: MySQL 8.x
- Authentication: JWT
- External APIs: OpenAI GPT (요약), AWS Polly (TTS), AWS S3 (파일 저장)
- Web Crawling: Jsoup 1.17.2
- Build Tool: Gradle

## 3. 1학기 이후 추가 구현 사항 (신규 개발)

### 3.1 네이버 뉴스 랭킹 크롤링 시스템 🆕
**구현 기간**: 2024년 2학기 초반
**주요 기능**:
- NaverRankingCrawler: 네이버 뉴스 랭킹 페이지 실시간 크롤링
- ArticleContentCrawler: 개별 기사 본문 수집
- 자동 스케줄러: 매 정시 5분마다 자동 크롤링 실행
- 6개 섹션별 랭킹 뉴스 수집 (정치~IT/과학)
- 중복 방지 로직 (URL 기준)
- 서버 부하 방지를 위한 1초 지연 처리

**기술적 특징**:
- Jsoup을 활용한 안정적인 웹 스크래핑
- Spring @Scheduled를 통한 자동화
- 에러 처리 및 재시도 로직 구현
- User-Agent 설정으로 크롤링 식별

### 3.2 뉴스 랭킹 API 시스템 🆕
**새로운 API 엔드포인트**:
- `GET /api/v1/news/ranking`: 섹션별/날짜별 랭킹 조회
- `GET /api/v1/news/ranking/top`: 최근 1위 뉴스 조회
- `GET /api/v1/news/ranking/{id}/content`: 특정 기사 본문 조회

**주요 기능**:
- 실시간 뉴스 랭킹 데이터 제공
- 섹션별 필터링 (정치, 경제, 사회, 생활/문화, 세계, IT/과학)
- 날짜별 히스토리 조회
- 상위 랭킹 뉴스 별도 API

### 3.3 데이터베이스 구조 개선 🆕
**새로운 테이블**:
- `news_rankings`: 네이버 뉴스 랭킹 데이터 저장
- 필드: id, section_id, press, rank, title, url, body, collected_at

**기존 테이블 개선**:
- `users` 테이블에서 email 필드 완전 제거
- username 기반 사용자 식별로 단순화
- 회원가입 시 관심사 동시 설정 기능

### 3.4 시스템 안정성 향상 🆕
**에러 처리 개선**:
- GlobalExceptionHandler 확장
- 크롤링 실패 시 재시도 로직
- 상세한 로깅 시스템

**성능 최적화**:
- 크롤링 지연 처리로 서버 부하 방지
- 데이터베이스 쿼리 최적화
- 메모리 사용량 제한 (본문 크기 2MB 제한)

### 3.5 배포 환경 구축 🆕
**Docker 컨테이너화**:
- Dockerfile 작성
- docker-compose.yml 설정
- nginx 설정

**AWS 인프라 스크립트**:
- EC2 자동 설정 스크립트 (ec2-setup.sh)
- RDS 설정 가이드 (rds-setup-guide.md)
- 인프라 구축 자동화 (aws-infrastructure-setup.sh)

## 4. 2학기 추가 개발 계획

- 관심분야 설정 인터페이스

**현재 상태**: OpenAI API 연동 구조 완성, 실제 API 키 설정 필요

- 요약 품질 평가 시스템

**현재 상태**: AWS Polly/S3 연동 구조 완성, API 키 설정 필요
**개선 계획**:

## 5. 기대 효과 및 차별점

### 5.1 기술적 차별점
- **실시간 뉴스 랭킹 크롤링**: 네이버 뉴스 랭킹의 실시간 데이터 수집
- **개인화 추천 시스템**: 사용자 관심사 기반 정확한 뉴스 필터링
- **AI 요약 + TTS 통합**: 읽기와 듣기를 모두 지원하는 멀티모달 서비스
- **견고한 백엔드 아키텍처**: Spring Boot 기반 확장 가능한 구조

### 5.2 사용자 경험 개선
- **시간 효율성**: AI 요약으로 빠른 뉴스 파악
- **접근성**: TTS 기능으로 시각적 제약 극복
- **개인화**: 관심분야 기반 맞춤형 뉴스 제공
- **편의성**: 웹/모바일 크로스 플랫폼 지원

### 5.3 확장 가능성
- **다양한 뉴스 소스 추가**: 현재 네이버 → 다른 포털 확장
- **소셜 기능**: 뉴스 공유, 댓글, 토론 기능
- **빅데이터 분석**: 사용자 행동 패턴 분석
- **B2B 서비스**: 기업용 뉴스 브리핑 서비스
