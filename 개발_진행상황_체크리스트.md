# Project TNP 개발 진행상황 체크리스트

## 프로젝트 개요
- **프로젝트명**: 개인 맞춤형 뉴스 요약 및 TTS 서비스
- **백엔드 기술스택**: Spring Boot 3.4.5, Java 21, MySQL 8.x, Spring Security 6.x
- **현재 상태**: MVP 기본 기능 구현 완료, TTS 기능 일부 비활성화

---

## ✅ 구현 완료된 기능

### 1. 사용자 인증 및 관리
- ✅ **회원가입** (`POST /api/signup`)
  - 유효성 검증 (아이디 4-20자, 비밀번호 8-20자 + 복잡도)
  - 관심분야 선택 기능 포함
- ✅ **로그인** (`POST /api/login`) 
  - JWT 토큰 발급
- ✅ **아이디 중복 체크** (`GET /api/auth/check-username`)
- ✅ **JWT 기반 인증 시스템**
  - JwtAuthenticationFilter 구현
  - Spring Security 설정 완료

### 2. 관심분야 관리
- ✅ **관심분야 목록 조회** (`GET /api/interests`)
- ✅ **사용자 관심분야 등록/수정** (`POST /api/user/interests`)
- ✅ **데이터베이스 설계**
  - `interest` 테이블 (마스터 데이터)
  - `user_interest` 테이블 (사용자-관심분야 연결)

### 3. 뉴스 서비스
- ✅ **개인화 뉴스 목록 조회** (`GET /api/news`)
  - 사용자 관심사 기반 필터링
  - 페이징 지원 (기본 10개/페이지)
- ✅ **뉴스 상세 조회** (`GET /api/news/{id}`)
- ✅ **뉴스 좋아요/취소** (`POST/DELETE /api/news/{id}/like`)
- ✅ **뉴스 북마크/취소** (`POST/DELETE /api/news/{id}/bookmark`)

### 4. 네이버 뉴스 랭킹 크롤링
- ✅ **네이버 뉴스 랭킹 크롤러** 구현
  - NaverRankingCrawler로 실시간 랭킹 수집
  - ArticleContentCrawler로 기사 본문 크롤링
- ✅ **스케줄러** (`@Scheduled`)
  - 매 정시 + 5분마다 자동 크롤링
- ✅ **랭킹 API**
  - `GET /api/v1/news/ranking` - 날짜/섹션별 랭킹 조회
  - `GET /api/v1/news/ranking/top` - 상위 랭킹 조회
  - `GET /api/v1/news/ranking/{id}/content` - 기사 본문 조회

### 5. AI 요약 서비스 (부분 구현)
- ✅ **OpenAI GPT 연동** 구현
  - Chat Completions API 사용
  - 한국어 3문장 이내 요약 프롬프트
- ✅ **요약 서비스 인프라** 준비
  - Summary 엔티티 및 Repository
  - SummaryService 인터페이스

### 6. 데이터베이스 설계
- ✅ **핵심 엔티티 구현**
  - User (사용자)
  - Interest (관심분야)
  - UserInterest (사용자-관심분야 연결)
  - News (뉴스)
  - NewsRanking (뉴스 랭킹)
  - Summary (요약)
  - ArticleLike (좋아요)
  - Bookmark (북마크)

### 7. 인프라 및 배포 준비
- ✅ **Docker 설정**
  - Dockerfile, docker-compose.yml
  - nginx 설정
- ✅ **AWS 인프라 스크립트**
  - EC2 설정 스크립트
  - RDS 설정 가이드
  - 배포 스크립트

## 🚀 즉시 활성화 가능한 기능

### 1. AI 요약 서비스
**필요한 작업**: OpenAI API 키만 설정
```properties
# application.properties에 추가
openai.api.key=실제_OpenAI_API_키
```

### 2. TTS 서비스
**필요한 작업**: AWS 자격증명 설정
```properties
# application.properties에 추가
cloud.aws.credentials.access-key=AWS_ACCESS_KEY
cloud.aws.credentials.secret-key=AWS_SECRET_KEY
cloud.aws.s3.bucket=S3_BUCKET_NAME
```
